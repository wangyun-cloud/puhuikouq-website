## Codebase Patterns
- Use system font stack (PingFang SC, Microsoft YaHei) instead of next/font/google to avoid network failures during static export builds
- shadcn CLI may install v4-style components on Next.js 14; ensure globals.css uses standard Tailwind v3 `@tailwind` directives
- Custom colors defined as CSS variables in globals.css and mapped in tailwind.config.ts
- Sanity Studio should be a separate package (`studio/`) because Sanity v5 requires React 19, which conflicts with Next.js 14's React 18
- Sanity project ID can be configured via `SANITY_STUDIO_PROJECT_ID` env var in `sanity.config.ts` and `sanity.cli.ts`
- Sanity schemas use `defineType` and `defineField` from `sanity` for type-safe schema definitions
- Sanity `createClient` throws if `projectId` is empty; initialize the client LAZILY in `lib/sanity.ts` to keep builds green before a project ID is configured
- `@base-ui/react` components use `render` prop for composition, NOT `asChild` like Radix (e.g., `<SheetTrigger render={<Button />} />`)
- Next.js 14 App Router natively supports `robots.ts` and `sitemap.ts` for static export—no extra library needed
